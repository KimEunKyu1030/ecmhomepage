노드

데이터 마이닝을 수행하기 위하여 데이터가 있어야 하며, 원천 데이터를 그대로 사용할 수 있는 경우는 거의 없기 때문에 데이터를 가공할 전처리 과정이 필요하고, 데이터의 통계적인 분석이나 분포 등을 알기 위하여 통계적 기법, 차팅(Charting) 등이 필요합니다. 또한, 데이터 마이닝의 핵심인 데이터 마이닝 알고리즘이 있어야 하고 이들의 결과를 볼 수 있는 기능도 필요합니다.

ECMiner™에서 노드란 앞에 언급한 데이터 마이닝의 요소 기능을 모듈화한 것입니다. ECMiner™의 노드는 사용 편의를 위하여 노드 기능의 유사성에 따라 7개의 범주로 나누어져 있습니다. 각 범주는 다음과 같습니다.
  • 입력 노드

    • 데이터 마이닝의 출발점인 데이터를 가져와 ECMiner™의 데이터 형식으로 변형하여 생성하는 기능을 갖습니다. ODBC 입력, OLEDB 입력, 액세스 데이터, 엑셀 데이터, 파일입력 등이 입력 노드에 해당합니다.

  • 전처리 노드

    • 초기 데이터는 유효한 데이터가 되기 위하여 적절한 가공이 필요합니다. "데이터 가공" 기능을 제공하는 것이 전처리 노드이며, 결측치 처리, 그룹화, 다중 파생변수, 변수순서, 병합, 분할, 선택, 열조합, 정렬, 채우기, 추가, 파생변수, 표본추출, 필터, 형태변경, RANKING 등이 전처리 노드에 해당합니다.

  • 차트 노드

    • 데이터를 시각화 하여 분석하기 위한 차팅 기능을 제공하는 노드입니다. 3차원차트, 관리도, 기본차트, 매트릭스 차트, 바차트, 박스차트, 컨튜어차트, 컨트롤차트, 통계차트, 파레토차트, 파이차트, 히스토그램 등이 차트 노드에 속합니다.

  • 모델링 노드

    • 데이터 마이닝의 핵심인 데이터 마이닝 알고리즘을 수행하는 노드입니다. ECMiner™에서는 15개의 데이터 마이닝 알고리즘을 지원하며 이들은 각각 APRIORI, CART, HIERACHICAL, KMEANS, KNN, LDA, LOGISTIC, MANUAL CART, MLP, MLR, PCA, PCR, PLS, QDA, RBF 노드로 대변됩니다.

  • 출력 노드

    • 생성된 결과를 보거나 저장하기 위한 기능을 제공합니다. 산출된 결과를 데이터베이스에 저장하기 위한 ODBC 출력, OLEDB 출력 노드와, 통계적 분석 결과를 보여주는 통계분석, 파일로 결과를 저장하기 위한 파일 출력, 결과의 피봇팅을 위한 피봇 노드, 화면에 결과를 나타내기 위한 화면표시 노드등이 출력 노드에 해당합니다.

  • 모델 평가 노드

    • 모델링 노드를 통하여 생성된 모델들을 비교, 평가하기 위한 기능을 제공합니다. ROC 차트, 모델평가, 이익도표 등이 이에 속합니다.

  • 모델 평가 노드

    • 생성된 모델은 모델링 노드의 결과물로 생성되는 노드입니다. 생성된 모델 노드를 이용하여 새로운 데이터에 대한 데이터 마이닝적인 관점에서의 새로운 결과를 산출할 수 있습니다.

ECMiner™의 노드는 모두 해당 작업을 수행하기 위한 속성을 갖습니다. 속성은 기본적으로 속성명과 속성값으로 구성됩니다. 예를 들어 파일입력 노드라면 어떤 파일을 읽어야 할 지에 대한 속성이 필요하고 이는 속성명(파일경로) 그리고 속성값으로 대변됩니다.

공통속성

ECMiner™의 노드는 ECMiner™에서 사용될 이름과 간단한 설명을 공통적으로 갖습니다.

속성그룹 속성명 설명
일반정보 이름 작업창에 표시될 노드의 이름이며 이 이름을 기본으로 메시지창 등에 노드 관련 메시지가 나타납니다. 노드에 기능에 따라 사용자가 알아 보기 쉽도록 이름을 짓는 것이 중요하며 필수 입력사항입니다.
설명 노드에 대한 간단한 설명을 입력하는 부분으로 선택 입력사항입니다. 설명이 입력된 경우 작업창에서 노드 위에 마우스를 올리면 입력한 설명이 나타납니다.
